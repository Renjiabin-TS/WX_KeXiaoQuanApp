# 定时邮件推送部署与测试指南

## ⚠️ 重要提醒

定时邮件推送功能需要手动部署到云端才能生效！

## 📋 修复记录

### 已修复的问题：

1. ✅ **定时触发器格式错误**
   - 修复前：`"0 0 23 * * * *"` (7个字段，不兼容)
   - 修复后：`"0 0 23 * * *"` (6个字段，标准格式)
   - 含义：每天23:00执行

2. ✅ **SMTP连接配置优化**
   - 增加超时时间（30秒 → 60秒）
   - 增加连接池大小（1 → 2）
   - 添加 TLS 版本限制
   - 增加最大消息数限制（防止连接被滥用）

## 🚀 部署步骤

### 步骤1：部署云函数

在微信开发者工具中：

1. 打开项目，在左侧目录中找到 `cloudfunctions/dailyEmailReport`
2. 右键点击 `dailyEmailReport` 文件夹
3. 选择 **「上传并部署：云端安装依赖」**
4. 等待部署完成（通常需要1-2分钟）

**注意：必须选择「云端安装依赖」，否则 nodemailer 无法正常工作！**

### 步骤2：验证定时触发器

部署完成后：

1. 打开微信开发者工具，点击顶部菜单「云开发」
2. 进入「云函数」页面
3. 找到 `dailyEmailReport` 云函数
4. 点击右侧「...」→「详情」
5. 查看「触发器」标签页，确认有以下触发器：

```
名称: dailyReportTrigger
类型: 定时触发器
配置: 0 0 23 * * *
```

**如果没有看到触发器，说明部署失败，需要重新部署！**

### 步骤3：手动测试

在云开发控制台测试：

1. 进入「云开发」→「云函数」
2. 找到 `dailyEmailReport`
3. 点击「测试」按钮
4. 点击「运行测试」
5. 查看返回结果和日志

**预期结果：**
```json
{
  "success": true,
  "message": "邮件推送完成"
}
```

### 步骤4：检查邮件

测试成功后：
1. 检查你的邮箱（注册时使用的邮箱）
2. 查看是否收到「课时统计日报」邮件
3. 如果没有收到，检查垃圾邮件文件夹

## 📊 邮件推送规则

### 触发时间
- **每天晚上23:00** 自动执行
- 包含本周统计、本月统计、当月明细

### 推送对象
- 所有在数据库中设置了 `email` 字段的用户
- 未设置邮箱的用户会被跳过

### 统计数据
- **本周统计**：从本周一到今天的课时数据
- **本月统计**：从本月1号到今天的课时数据
- **当月明细**：本月所有课时记录的详细列表

## 🔧 故障排查

### 问题1：云函数部署失败

**错误提示：** 依赖安装失败

**解决方法：**
1. 检查网络连接
2. 删除 `cloudfunctions/dailyEmailReport` 目录下的 `node_modules` 文件夹（如果存在）
3. 重新选择「上传并部署：云端安装依赖」

### 问题2：定时器不触发

**检查步骤：**
1. 确认云函数已成功部署
2. 确认触发器配置正确（在云开发控制台查看）
3. 查看云函数日志，在23:00后是否有执行记录
4. 如果没有日志，说明触发器未生效，需要重新部署

### 问题3：邮件发送失败

**错误提示：** 邮件发送失败

**可能原因：**
1. 163邮箱授权码已过期
2. SMTP连接超时
3. 邮箱地址错误

**解决方法：**
1. 登录163邮箱，重新生成授权码
2. 更新 `index.js` 中的授权码
3. 重新部署云函数
4. 查看云函数日志中的详细错误信息

### 问题4：收不到邮件

**检查步骤：**
1. 检查垃圾邮件文件夹
2. 确认数据库中用户的 `email` 字段值正确
3. 在云开发控制台手动触发测试
4. 查看云函数日志，确认邮件是否成功发送

## 📝 配置修改

### 修改推送时间

编辑 `cloudfunctions/dailyEmailReport/config.json`：

```json
{
  "triggers": [
    {
      "name": "dailyReportTrigger",
      "type": "timer",
      "config": "0 0 20 * * *"  // 改为每天20:00
    }
  ]
}
```

**修改后必须重新部署云函数！**

### 修改邮件内容

编辑 `cloudfunctions/dailyEmailReport/index.js` 中的 `generateEmailHTML` 函数。

### 修改SMTP配置

编辑 `cloudfunctions/dailyEmailReport/index.js` 中的 `getTransporter` 函数。

## 🚨 常见错误及解决方案

### 错误1：`ERR_CONNECTION_TIMEOUT`

**原因：** SMTP连接超时

**解决：** 已在代码中增加超时时间到60秒，如果仍失败，检查网络或更换邮箱服务商

### 错误2：`Invalid login`

**原因：** 授权码错误或过期

**解决：** 登录163邮箱重新生成授权码，更新代码后重新部署

### 错误3：`nodemailer is not defined`

**原因：** 依赖未安装

**解决：** 必须选择「云端安装依赖」部署模式
