# 每日定时邮件推送功能说明

## 功能概述

系统会在每天晚上11点（23:00）自动向所有已登录用户发送课时统计邮件，内容包括：

1. **本周课时统计**
   - 总课时
   - 一对一课时
   - 班课课时

2. **本月课时统计**
   - 总课时
   - 一对一课时
   - 班课课时

3. **当月明细信息**
   - 每条课时的详细记录（日期、类型、时间、学生/班级、课时）

## 配置步骤

### 1. 部署云函数

在微信开发者工具中：

1. 右键点击 `cloudfunctions/dailyEmailReport` 目录
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成

### 2. 配置定时触发器

云函数已配置定时触发器，规则为：

```json
"0 0 23 * * * *"
```

表示：每天23:00:00执行一次（秒 分 时 日 月 周）

### 3. 确认用户邮箱

确保用户在个人中心设置了正确的邮箱地址：

1. 打开小程序
2. 进入「个人」页面
3. 点击「设置邮箱」
4. 输入邮箱地址并验证

## 邮件配置

当前邮件发送配置：

- **SMTP 服务器**：smtp.163.com
- **端口**：465 (SSL)
- **发件人邮箱**：15369581103@163.com
- **发件人名称**：课时管理系统

### 注意事项

1. **163邮箱限制**：网易邮箱有每日发送限制（通常为200封），如果用户数量较多，建议使用企业邮箱
2. **垃圾邮件风险**：建议用户将发件人邮箱添加到白名单
3. **发送失败处理**：如果发送失败，会在云函数日志中记录错误信息

## 自定义配置

### 修改推送时间

编辑 `cloudfunctions/dailyEmailReport/config.json` 文件：

```json
{
  "triggers": [
    {
      "name": "dailyReportTrigger",
      "type": "timer",
      "config": "0 0 20 * * * *"  // 改为每天20:00推送
    }
  ]
}
```

修改后需要重新部署云函数。

### 修改邮件内容

编辑 `cloudfunctions/dailyEmailReport/index.js` 文件中的 `generateEmailHTML` 函数。

## 故障排查

### 问题1：邮件没有收到

**可能原因**：
1. 用户未设置邮箱地址
2. 邮箱地址错误
3. 邮件被误判为垃圾邮件
4. 云函数执行失败

**解决方法**：
1. 检查云函数日志是否有错误
2. 确认用户邮箱地址是否正确
3. 检查垃圾邮件文件夹
4. 尝试手动触发测试

### 问题2：云函数执行失败

**检查步骤**：
1. 查看云函数日志中的错误信息
2. 确认依赖包是否正确安装
3. 检查数据库权限配置
4. 确认定时触发器是否正确配置

### 问题3：邮件内容不正确

**检查步骤**：
1. 确认数据库中课时数据是否正确
2. 检查时间范围计算逻辑
3. 查看云函数日志中的数据输出

## 监控与维护

### 定期检查

建议每周检查以下内容：

1. 云函数执行日志
2. 邮件发送成功率
3. 用户邮箱地址有效性
4. 数据库查询性能

### 性能优化

如果用户数量较多，可以考虑：

1. 批量发送邮件，避免频繁连接SMTP服务器
2. 添加发送失败重试机制
3. 优化数据库查询，使用索引
4. 考虑使用消息队列异步处理

## 常见问题

**Q: 如何临时暂停邮件推送？**

A: 可以在云开发控制台暂时禁用定时触发器，或者注释掉 config.json 中的 triggers 配置后重新部署。

**Q: 可以为不同用户设置不同的推送时间吗？**

A: 当前实现是统一时间推送所有用户。如需差异化推送，需要修改云函数逻辑，为每个用户单独设置推送时间。

**Q: 邮件支持哪些格式？**

A: 当前使用 HTML 格式，支持富文本展示。如需纯文本格式，可以修改邮件内容生成逻辑。

**Q: 如何修改邮件主题？**

A: 在 `index.js` 文件中修改 `mailOptions.subject` 字段即可。
